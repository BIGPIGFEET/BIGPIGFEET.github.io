# Segment Anything Model 2 (SAM 2)

> 论文：Ravi et al., 2024, "SAM 2: Segment Anything in Images and Videos", Meta AI
> 论文链接：https://arxiv.org/abs/2408.00714

## 背景

SAM在图像分割上取得了成功，但视频分割仍面临挑战。SAM 2将SAM的能力扩展到**图像和视频统一分割**。

## 核心贡献

### 1. 统一架构

SAM 2使用统一的架构同时处理图像和视频分割：

- **图像编码器**：ViT-H/ViT-L（与SAM相同）
- **视频编码器**：添加了记忆模块（Memory机制）
- **流式推理**：支持在线和离线视频处理

### 2. Memory Mechanism（记忆机制）

视频分割的关键创新：

```
当前帧 → Memory Bank → 下一帧预测
```

- **Memory Encoder**：将当前预测编码为记忆
- **Memory Attention**：在处理后续帧时检索相关记忆
- **Memory Bank**：存储多帧的历史信息

### 3. Promptable Video Segmentation

SAM 2将视频视为帧序列，支持：

- 空间提示（点、框）：指定要跟踪的物体
- 时间提示（点击）：在任意帧指定，传播到整个视频

### 4. Interactive Refinement

通过点击交互式精炼分割结果：

- 正点击：扩展掩码
- 负点击：收缩掩码
- 自动传播修正到所有帧

## 关键技术

### 遮挡处理

SAM 2通过记忆机制处理物体遮挡：

- 短期记忆：最近几帧的分割结果
- 长期记忆：关键帧的语义信息
- 重新出现时能重新定位目标

### 跨物体歧义

当多个相似物体出现时，通过：

- 实例ID区分不同物体
- 点击提示指定具体目标
- 记忆机制保持目标一致性

## 性能

SAM 2在多个基准上达到SOTA：

- **DAVIS Video Object Segmentation**：显著提升
- **SAM-VOS**：视频目标分割
- **UVO**：开放世界视频分割

## 优势与局限

**优势**：
- 统一的图像/视频分割
- 实时处理能力
- 强大的zero-shot能力
- 处理遮挡和重识别

**局限**：
- 计算资源需求较高
- 某些复杂场景仍会失败
- 视频编码器增加了延迟

## 与SAM的区别

| 特性 | SAM | SAM 2 |
|------|-----|-------|
| 任务 | 图像分割 | 图像+视频分割 |
| 架构 | 图像编码器 | 图像编码器+视频记忆模块 |
| 提示 | 空间提示 | 空间+时间提示 |
| 输出 | 分割掩码 | 分割掩码+实例ID |

## 启发

1. **记忆机制是关键**：视频理解需要时序记忆
2. **统一范式**：图像和视频可以共享模型能力
3. **交互式迭代**：人机协同提升分割质量
